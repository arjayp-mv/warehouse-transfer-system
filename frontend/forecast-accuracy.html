Monthly Workflow Guide

Follow these steps each month to maintain accurate forecasts and continuous improvement:

Step 1: Upload Sales Data
  • When: 1st-2nd of each month
  • What: Upload previous month's sales to monthly_sales table
  • Include: Burnaby sales, Kentucky sales, stockout days
  • Verify: Check data uploaded correctly before proceeding
Step 2: Update Accuracy
  • When: After Step 1 completes
  • How: Select completed month, click "Update Accuracy"
  • Check: Review success message for forecasts updated
  • Warning: Do NOT run for current/future months
Step 3: Review Dashboard
  • Check Overall MAPE (target: under 20%)
  • Review 6-month trend for improvements
  • Examine ABC/XYZ heatmap for problem areas
  • Identify Problem SKUs requiring attention
Step 4: Run Learning Analysis
  • When: After reviewing dashboard (2nd-3rd of month)
  • Requirement: Need 2-3 months of accuracy data
  • How: Click "Run Learning Analysis" button
  • Output: Generates adjustment recommendations
Step 5: Apply Adjustments
  • Review recommendations in learning_adjustments table
  • Apply growth rate adjustments for problem SKUs
  • Consider method changes for consistent underperformers
  • Update seasonal factors if patterns detected
Important: Never run these operations with test data or for future months - this will corrupt your historical metrics and learning system.
Total time required: 15-30 minutes per month | Best performed: 1st week of each month
